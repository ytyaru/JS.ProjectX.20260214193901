# Classification TODO (文書分類の網羅性検証計画)

本プロジェクトが扱うべき「文書」の範囲を確定させるため、論理的なパターンと現実に存在する文書種別を照合し、分類の妥当性を検証する手順書。

## 目的
「3成分（真偽・論理・情緒）× 指向（自・他・両）」のマトリクスが、世の中の文書を漏れなく、かつ矛盾なく分類できるかを証明する。あるいは、分類不能なケース（破綻）を発見する。

## 前提条件：分類軸の定義

### 1. 成分 (Component) - 12パターン
*   **真偽性 (Truthfulness):** 事実 / 虚構 / 混合
*   **論理 (Logicality):** 有 / 無
*   **情緒 (Emotionality):** 有 / 無
*   *(計算式: 3 × 2 × 2 = 12)*

### 2. 指向 (Direction) - 3パターン
*   **自己 (Self):** 自分のために書く（内省、記録）。
*   **他者 (Others):** 他人のために書く（伝達、操作）。
*   **両方 (Both):** 自分と他者の両方を意識する（公開された私記）。
*   *(注: 機械向けは「開発者」の領分とし、執筆者からは除外)*

### 3. 総パターン数
*   **36パターン** (12 × 3)

---

## 実行プロセス

以下の順序でリストを作成し、マトリクスへのマッピングを行う。

### Step 1: パターン網羅表の作成
36個の空の箱を用意する。
*   行: 成分 (12種)
*   列: 指向 (3種)

### Step 2: 文書・文章種別の洗い出し (Listing)
分類表に当てはめる前に、現実に存在する名称を可能な限り列挙する。

*   **リストA: 文章種別 (Text Types)**
    *   記録文、論文、意見文、叙述文、描写文、韻文、命令文、定義文、挨拶文、始末書...
*   **リストB: 文書種別 (Document Types)**
    *   新聞、雑誌、小説、詩集、日記、ブログ、仕様書、契約書、チラシ、手紙、教科書、辞書、脚本、演説原稿...

### Step 3: マッピングと検証 (Mapping & Verification)
Step 2で挙げた項目を、Step 1の表に配置していく。

*   **検証観点:**
    1.  **配置不能:** どの箱にも入らない文書はあるか？（軸の不足）
    2.  **複数該当:** 一つの文書が複数の箱にまたがる場合、それは「混合」で説明つくか、それとも定義が曖昧か？
    3.  **空箱:** 現実には存在しない（価値がない）組み合わせはあるか？（例: 「虚構・無論理・無情緒・他者向け」は成立するか？）

### Step 4: 結論の導出
検証結果に基づき、分類軸の修正、または本プロジェクトの守備範囲（スコープ）の確定を行う。
