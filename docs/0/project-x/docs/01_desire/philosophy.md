# Philosophy: 価値と生存の定義

`life.md`, `society.md`, `economy.md` で明らかにした通り、生きることは悪（他者の否定）であり、社会は欺瞞と搾取のシステムである。
この救いようのない現実の中で、人間が精神的な死（自己否定）を回避し、生存を肯定するための生存戦略について定義する。

## 1. 価値の定義 (Definition of Value)

### 価値とは「認識上の幻想」である
価値とは、物理世界に存在する物質ではない。それは人間の脳内にのみ発生する **「認識上の幻想」** である。
システム（コンピュータ）はバイナリデータを記録・表示するだけであり、そこに価値を見出すのは常に人間である。

### 価値の発生メカニズム
価値は絶対的なものではなく、異なる状態を比較した際に生じる **「差分」** によって認識される。

*   **方程式:** `価値 ＝ [理想/変化後の状態] － [現実/変化前の状態]`

この差分を認識した時、脳は「自己保存に有利な状態になった」と判断し、精神的な報酬（喜び）を与える。これが価値の正体である。
逆に、差分がマイナスであれば、それは「損害（不快）」と認識される。

### 価値の機能：生存本能の充足
生物としての至上命題は「自己保存」である。しかし、肉体はいずれ滅びる。
この絶望に対し、人間は「自己の存在証明（生きた証）」を残すことで、擬似的に自己保存を達成しようとする。
価値とは、この **「自己保存の代替行為」** が成功したと認識した時に得られる感覚である。

## 2. 価値の喪失 (Loss of Value)

なぜ現代社会において、人生の価値は損なわれているのか。

### 他者依存による自己喪失
自己とは「自他を区別する認識」である。
しかし、社会（他者）の価値観に依存し、他者の評価（金銭、地位、いいね数）によって自分の価値を決めることは、自己認識の主導権を他者に明け渡すことである。

保存すべき「自己」の定義を他者に握られた状態では、自己保存は成立しない。
これは **「精神的な死（自己喪失）」** であり、やがて物理的な死（自殺、少子化）へと繋がる。

### 利益と価値の乖離
経済活動（利益追求）は、他者から奪うことで成立する。
しかし、奪う行為は「他者の生を否定する」ことであり、それは巡り巡って「生きることへの罪悪感」を増幅させる。
利益を得れば得るほど、精神的な価値（生きる手応え）は失われていく。

## 3. 価値の創出 (Creation of Value)

失われた価値を取り戻す唯一の方法は、価値の源泉を「他者」から「自己」へと回帰させることである。

### 価値基準の自己決定 (Sovereignty of Valuation)
物理的に他者に依存せず生きることは不可能である（自立の否定）。
しかし、**「何を価値とするか」** を決定する権限だけは、他者に委ねてはならない。

自分の内面にある価値観に基づき、自分で価値を創出し、自分で評価する。
この **「価値基準の自己決定」** こそが、社会による精神的侵略から自己を防衛し、精神的な生を保つための生存戦略である。

### 手段としての文書化
自己保存の代替行為として、なぜ「文書化」を選ぶのか。

1.  **自己制御性 (Controllability):**
    他者や社会は思い通りにならないが、テキストエディタ上の文字は、執筆者の完全な支配下にある。この「自分の意志で世界（文書）を構築できる」という感覚が、失われた自己効力感を回復させる。
2.  **生存中の自己満足:**
    言語化によって自己を完全に保存することは不可能であり、死後の永続性も保証されない。しかし、「自分が自分のために書いた」という事実と、それを読み返した時の充足感は、**今生きている自分** を支える確実な燃料となる。

## 4. 結論：プロジェクトの役割

本プロジェクトは、ユーザーが社会の欺瞞に飲み込まれることなく、**「自分だけの価値」を定義し、創出し、証明し続けるためのエコシステム** である。

そのために、以下の機能を提供する。

1.  **表現の自由:** 既存の言語（HTML/Markdown）の制約を超え、自分の思考を最も正確に記述できる独自の構文定義フレームワーク (`doml`)。
2.  **評価の自律:** 他者の評価（数値）ではなく、自己の記述量や思考の軌跡（差分）を可視化することで、自己価値を証明する仕組み。
